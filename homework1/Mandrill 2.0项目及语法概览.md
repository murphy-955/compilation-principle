# Mandrill (山魈) v2.0 语言参考手册

Mandrill 是一门为编译原理课程设计的命令式编程语言。v2.0 版本在 v1.0 极简主义的基础上，引入了动态内存管理、UTF-32 字符串支持以及完整的函数调用机制。

---

## 1. 词法结构 (Lexical Structure)

### 1.1 字符集
Mandrill 源代码使用 **UTF-8** 编码。
* **标识符字符**：小写字母 `a-z`、大写字母 `A-Z`、下划线 `_`。
* **数字**：`0-9`。
* **运算符与分隔符**：`+ - * / % < > = ! ( ) [ ] { } , ; ' "`。
* **不可见字符**：空格、换行、回车、制表符仅用于分割词素。

### 1.2 注释
使用 `#` 表示行注释，自 `#` 起至行尾的内容将被忽略。

### 1.3 词素 (Tokens)
* **关键字**：`if`, `else`, `while`, `read`, `put`, `write`, `get`, `func`, `global`, `local`, `return`, `break`, `continue`。
* **整数常量**：十进制表示，不设负数（通过 `0 - x` 获得），长度不超过 8 位。
* **字符常量**：单引号围绕，支持 `\n`, `\\`, `\'`。
* **字符串常量**：双引号围绕。在编译时，字符串被转换为 **UTF-32** 编码（每个字符 4 字节），并以 `0` 结尾。（C风格字符串，只不过是UTF-32的）

## 2. 数据类型与变量

### 2.1 类型系统
Mandrill 只有两种类型，变量类型通过**首次出现的形式**推断：
1.  **整数 (Integer)**：32 位有符号整数。
2.  **数组 (Array)**：本质为指向堆内存的指针。

### 2.2 作用域 (Scope)
* **全局变量**：在函数外定义，或使用 `global` 显式声明。未经声明且直接使用的变量默认为全局。
* **局部变量**：在函数内通过 `local` 声明。可见域从声明处开始，到所属的代码块 `{}` 结束。

---

## 3. 表达式与运算符

运算符优先级从高到低排列如下：

| 优先级 | 运算符 | 描述 | 结合性 |
| :--- | :--- | :--- | :--- |
| 1 | `( )`, `[ ]` | 括号、数组下标 | 从左到右 |
| 2 | `*`, `/`, `%` | 乘、除、取模 | 从左到右 |
| 3 | `+`, `-` | 加、减 | 从左到右 |
| 4 | `<`, `<=`, `>`, `>=` | 关系比较 | 从左到右 |
| 5 | `==`, `!=` | 相等判断 | 从左到右 |

**注意**：不支持逻辑运算（用关系运算代替）和位运算。负号 `-exp` 需写为 `(0 - exp)`。

---

## 4. 语句 (Statements)

### 4.1 赋值与内存申请
* **数组声明**：`a[] = 10;` 表示申请 10 个单位（每个 4 字节）的堆空间。
* **字符串赋值**：`s[] = "Hello";` 自动根据字符串长度申请空间并初始化。

### 4.2 控制流
* **条件语句**：`if (exp) { ... } else { ... }`，`else` 部分不可省略。
* **循环语句**：`while (exp) { ... }`，花括号必须明确指明。
* **跳转**：
    * `break;`：跳出最内层循环。
    * `continue;`：跳过本次循环剩余部分。

---

## 5. 函数 (Functions)

### 5.1 定义语法
* **返回整数**：`func name(a, b, c[], d) { ... }`。
* **返回数组**：`func[] name(a, b, c[], d) { ... }`。
* **返回语句**：`return exp;`。若未指定，整数函数返回 `0`。

### 5.2 参数传递
所有参数均为值传递。但由于数组变量本身是地址，传递数组参数相当于**引用传递**。

---

## 6. 特殊输入输出 (I/O)

Mandrill 通过特殊关键字实现 I/O，其行为取决于变量类型：

* **`write`**：
    * `write = 整数;`：以十进制格式输出数值。
    * `write = 数组;`：将数组内容作为 UTF-32 字符串输出（考虑到多数终端与文件编码为UTF-8，故这里需要进行一个编码转换）。
* **`read`**（始终触发动态内存申请）：
    * **首次出现**：`a[] = read;` 声明 `a` 为数组并读入字符串。
    * **覆盖赋值**：`a = read;` 如果`a`是数组则释放 `a` 原有的空间，申请新空间存入新字符串。否则从stdin读入一个整数。
* **`get` / `put`**：读取或输出单个字符，读入需从UTF-8转换为UTF-32，输出则需要把UTF-32转换为UTF-8。

---

## 7. 语言限制与未定义行为
* **数组维度**：原生仅支持一维数组。
* **数组越界**：不进行运行时边界检查，结果未定义。
* **内存回收**：`a[] = read` 需手动或由 VM 处理旧空间的释放。
* **语句上限**：单个语句序列建议不超过 256 条语句。
